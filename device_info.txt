If you already have:

* Screenshot capture ✅
* Cropping status/navigation bars ✅

Then the simplest and most robust solution is **SSIM + Difference Highlighting**.

```python
import cv2
from skimage.metrics import structural_similarity as ssim

# Load images
img1 = cv2.imread("reference.png")
img2 = cv2.imread("test.png")

# Resize if needed
img2 = cv2.resize(img2, (img1.shape[1], img1.shape[0]))

# Convert to grayscale
gray1 = cv2.cvtColor(img1, cv2.COLOR_BGR2GRAY)
gray2 = cv2.cvtColor(img2, cv2.COLOR_BGR2GRAY)

# Calculate SSIM
score, diff = ssim(gray1, gray2, full=True)

print(f"Similarity Score: {score:.4f}")

# Convert diff image
diff = (diff * 255).astype("uint8")

# Threshold differences
thresh = cv2.threshold(
    diff,
    0,
    255,
    cv2.THRESH_BINARY_INV | cv2.THRESH_OTSU
)[1]

# Find contours
contours, _ = cv2.findContours(
    thresh,
    cv2.RETR_EXTERNAL,
    cv2.CHAIN_APPROX_SIMPLE
)

# Draw boxes around significant differences
for c in contours:
    area = cv2.contourArea(c)

    # Ignore tiny differences
    if area < 200:
        continue

    x, y, w, h = cv2.boundingRect(c)

    cv2.rectangle(
        img2,
        (x, y),
        (x + w, y + h),
        (0, 0, 255),
        2
    )

# Save result
cv2.imwrite("comparison_result.png", img2)

# Pass / Fail logic
THRESHOLD = 0.95

if score >= THRESHOLD:
    print("PASS - UI matches")
else:
    print("FAIL - UI differs")

print(f"Final Similarity: {score:.4f}")
```

### Better Version (Recommended)

Before SSIM, apply a slight blur to reduce false positives from font rendering differences:

```python
gray1 = cv2.GaussianBlur(gray1, (5,5), 0)
gray2 = cv2.GaussianBlur(gray2, (5,5), 0)
```

### Suggested Thresholds

```text
0.98 - 1.00  → Identical
0.95 - 0.98  → Very similar
0.90 - 0.95  → Minor UI changes
< 0.90       → Significant differences
```

If your goal is to compare the same app screen across different Android phones (Samsung, OnePlus, Xiaomi, Pixel, etc.), I would use:

```python
THRESHOLD = 0.93
MIN_CONTOUR_AREA = 500
GAUSSIAN_BLUR = (5,5)
```

This combination usually ignores tiny rendering differences while still detecting real UI layout changes.

Yes. In fact, for smartphone UI comparison, SSIM is only one option. Depending on your requirements, there are better alternatives:

### 1. Perceptual Hashing (pHash) — Fast and Simple

Instead of comparing pixels, convert each screenshot into a perceptual fingerprint.

```python
from PIL import Image
import imagehash

hash1 = imagehash.phash(Image.open("reference.png"))
hash2 = imagehash.phash(Image.open("test.png"))

distance = hash1 - hash2

print("Hash Distance:", distance)
```

Typical interpretation:

```text
0-5    -> Almost identical
5-15   -> Minor differences
15-30  -> Noticeable changes
30+    -> Different screens
```

Advantages:

* Ignores minor rendering differences.
* Very fast.
* Easy to implement.

Disadvantage:

* Doesn't tell you where differences are.

---

### 2. ORB Feature Matching (Very Good for UI)

OpenCV's ORB compares visual features rather than pixels.

```python
import cv2

img1 = cv2.imread("reference.png", 0)
img2 = cv2.imread("test.png", 0)

orb = cv2.ORB_create(5000)

kp1, des1 = orb.detectAndCompute(img1, None)
kp2, des2 = orb.detectAndCompute(img2, None)

bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)

matches = bf.match(des1, des2)

matches = sorted(matches, key=lambda x: x.distance)

similarity = len(matches) / max(len(kp1), len(kp2))

print("Similarity:", similarity)
```

Advantages:

* More robust to resolution differences.
* Handles slight shifts and scaling.
* No OCR required.

---

### 3. Template-by-Regions

Divide the screen into blocks:

```text
Header
Content Area
Bottom Section
```

Compare each block separately.

Example:

```python
header_ref = img1[0:200,:]
header_test = img2[0:200,:]

content_ref = img1[200:1500,:]
content_test = img2[200:1500,:]
```

Then run SSIM on each region.

This helps identify exactly which section changed.

---

### 4. Deep Learning Embeddings (Best Accuracy)

Use a pretrained model such as:

* MobileNetV3
* EfficientNet
* DINOv2
* CLIP

Extract feature vectors and compare with cosine similarity.

```python
similarity = cosine_similarity(feature1, feature2)
```

Advantages:

* Ignores clock changes.
* Ignores small icon changes.
* Focuses on overall layout.

Disadvantage:

* Requires PyTorch/TensorFlow.

---

### 5. Hybrid Method (What I Would Use)

```text
Crop status bar
Crop navigation bar
      ↓
pHash check
      ↓
ORB feature matching
      ↓
SSIM verification
      ↓
Final score
```

For example:

```text
Final Score =
40% ORB +
40% SSIM +
20% pHash
```

For a college project or practical UI regression tool, **ORB + SSIM together** is usually the best balance of accuracy, speed, and simplicity without OCR or heavy AI models. It handles changing clocks, battery indicators, and small rendering differences much better than plain OpenCV image comparison.
